Infinispan Ensemble

Infinispan Ensemble は Infinispan API を公開し、データ・グリッド・クラスターの集合を連合させます。データ・グリッドの集合は、クライアントからは、Infinispan Ensemble が以下のユーザ定義の保証を提供する単一のデータ・グリッドとして認識されます。

  • 地理的保証：あるデータの集合が、地理的に離れた2つのクラスターに重複して存在することが保証されます。
  • 信頼性：あるデータの集合が少なくともn個のクラスターに存在することが保証されます。

ユーザーは、Key/Value APIやM/R APIを含む一連のInfinispan APIを使ってデータにアクセスできます。このプロジェクトは、ビジネスで定義された依存性や法的な保証（例えば、領土の制約）を保証しながら、低遅延、ローカルまたはニアローカルな操作への扉を開きます。

このプロジェクトは、Neuchatel大学がLEADSプロジェクトに参加することで貢献しています。

詳細は、このプロジェクトのGitHubリポジトリに掲載されています。

Infinispan Atomic Factory

分散システムでは、故障や非同期の可能性がある多数の異種コンポーネントが集約されています。このような気まぐれな性質を制御するために、システム設計者は、ステートマシンレプリケーションのようなノンブロッキング技術に頼っています。この手法は、選抜されたグループのマシンで複製された共有オブジェクトに一貫したノンブロッキング操作を提供します。Atomic Object Factoryは、Infinispan上でステートマシンレプリケーションの枠組みを実装したものです。ファクトリーの使い方は、Javaのsynchronizedキーワードを使うのと同じくらい簡単です。Serializableクラスでファクトリーを呼び出すだけで、複数のInfinispanサーバー上でインスタンス化されたオブジェクトの依存性、一貫性、生きていることが保証されます。ファクトリーは、Infinispanのキャッシュ上に任意の（シリアライズ可能な）クラスのオブジェクトをインスタンス化することができるという意味で普遍的なものであり、同時アクセスにもかかわらず強力な一貫性を確保しつつ、透過的にオブジェクトを複製し、耐久性を持たせられます。

このプロジェクトは、従来Apache ZooKeeperなどのプロジェクトが提供していたInfinispanの新しいユースケースへの扉を開くものです。

このプロジェクトは、Neuchatel大学のLEADSプロジェクトへの参加によるものです。

詳細は、このプロジェクトのGitHubリポジトリでご覧いただけます。

InfinispanでのApache Gora APIの使用

Apache Goraビッグデータで永続化の抽象化APIでマップ・リデュース操作を実行します。

LEADSプロジェクトへの貢献を通じて、Neuchatel大学はInfinispanでデータを永続化するApache Goraバックエンドを提供しています。
これは、Apache Goraを使用しているプロジェクトを利用して、データを永続化し、マップ/リデュースジョブを実行したい場合に特に有効です。主な例としては、Apache Nutch、これは非常に人気があり効率的な Web クローラです。 NutchのデータをInfinispanにバックアップされたデータグリッドに保存し、各ノードでM/Rジョブを実行できます。

次のステップは？

Infinispan Goraの動作を確認するには、Unicrawlのチュートリアルに従うのが良いでしょう。このチュートリアルでは、Infinispanの小さなクラスタで、Apache NutchのデータをApache Goraを通して保存し、InfinispanのAvroシリアライゼーションを使用しています。チュートリアルをご覧ください。

コードや詳細な情報は寄与されたGitHubリポジトリでご覧いただけます。

Apache Avroによるデータのシリアライズ

Apache Avroは、効率的でコンパクトなシリアル化フォーマットです。

LEADSプロジェクトへの貢献を通じて、Neuchatel大学はHotRodプロトコルを介してAvroで定義された型を保存、取得、照会するためのAvroバックエンドを提供しています。
このバックエンドは、このポータブルなフォーマットでデータを保存したい場合や、Avroを使用するアプリケーションとデータを共有したい場合に特に役立ちます。
